웹은 HTML과 CSS, JavaScript를 기반으로 구축됩니다. 수 년간 많은 웹 개발자들은 컨텐츠는 HTML로, 디자인은 CSS로, 로직을 JavaScript로 작성하여 별도의 파일에 코드를 보관하였습니다.

<br />

하지만, 시간이 흐르면서 웹에서 상호작용의 중요성이 점점 증가하고 있습니다. 유저와의 상호작용이 컨텐츠를 결정하는 수준까지 도달하였습니다. 이는 JavaScript 로직을 이용해 HTML을 수정해야하는 빈도가 크게 증가했다는 의미이기도 합니다.

<br />

리액트는 컴포넌트와 JSX를 이용하여 이러한 문제를 쉽개 해결합니다. 컴포넌트는 JavaScript 로직과 HTML 마크업을 모두 포함하여, 로직이 쉽게 마크업을 변경할 수 있도록 만들어줍니다.

<br />

컴포넌트가 표현하는 HTML 마크업은 JSX를 이용해 작성합니다. JavaScript를 이용해 HTML을 변경하는 방법은 코드가 매우 길어져 복잡한 웹 애플리케이션을 구축할 때 유지보수가 힘들 수 있습니다. 하지만, JSX를 사용한다면 짧고 직관적인 코드를 사용하여 생산성을 향상시킬 수 있습니다.

# 명령형 프로그래밍과 선언형 프로그래밍

JavaScript를 이용해 HTML을 직접 업데이트하는 방식이 대표적인 명령형 프로그래밍이라고 할 수 있습니다.

```html
<form>
  <input type="text" name="name" />
  <p id="fail" style="color: red; display: none">Something went wrong.</p>
  <p id="success" style="color: blue; display: none">Success.</p>
  <button>Submit</button>
</form>
```

```js
const $form = document.querySelector("form");
const $input = document.querySelector("input");
const $button = document.querySelector("button");
const $errorMessage = document.querySelector("#fail");
const $successMessage = document.querySelector("#success");

function hide(node) {
  node.style.display = "none";
}

function show(node) {
  node.style.display = "block";
}

$form.addEventListener("submit", (e) => {
  e.preventDefault();
  let state = Math.random();

  if (state > 0.5) {
    hide($input);
    hide($button);
    hide($errorMessage);
    show($successMessage);
  } else {
    show($input);
    show($button);
    hide($successMessage);
    show($errorMessage);
  }
});
```

웹 애플리케이션이 복잡해질수록 필요한 변수와 함수가 훨씬 많아져 유지보수가 매우 힘들어진다는 단점이 존재합니다. 이러한 명령형 프로그래밍의 단점을 해결하기 위해 등장한 방법이 바로 선언형 프로그래밍입니다. 선언형 프로그래밍은 JavaScript를 이용해 HTML을 하나씩 바꾸는 방법이 아니라, 내가 원하는 HTML의 모습을 보여주고, 이렇게 바꿔달라고 선언하는 방법입니다.

<br />

선언형 프로그래밍을 사용하는 대표적인 라이브러리가 바로 React입니다. JSX를 이용해 내가 원하는 마크업의 모습을 그리고, React에게 마크업을 이렇게 바꿔달라고 요청하면, React는 여러가지 연산 후, HTML의 형태를 바꿉니다.
